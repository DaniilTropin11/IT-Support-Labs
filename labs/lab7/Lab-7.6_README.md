# 🧪 Лабораторная работа №7.6

**Тема:** Интерфейс Zabbix и базовые объекты системы мониторинга

------------------------------------------------------------------------

## 🎯 Цель работы

Познакомиться с интерфейсом Zabbix и изучить основные объекты системы
мониторинга: hosts, templates, items, triggers, tags и macros, а также
научиться анализировать проблемы и данные мониторинга.

------------------------------------------------------------------------

## 📚 Задания

1.  Просмотрите внимательно видео курса:

https://www.youtube.com/watch?v=vfYrCVIEJ_4&list=PLdQohrQ3OmqS2wo9MwFKJtnXcVH5SCOh8&index=7

2.  Изучите структуру интерфейса Zabbix.

Найдите и опишите назначение следующих разделов:

-   Monitoring
-   Problems
-   Latest data
-   Configuration → Hosts
-   Configuration → Templates

3.  Изучите структуру хоста в Zabbix.

Найдите и изучите параметры хоста:

-   Host groups
-   Interfaces
-   Tags
-   Macros
-   Inventory

4.  Изучите использование шаблонов (Templates).

Определите:

-   какие шаблоны применены к хостам
-   какие items и triggers наследуются от шаблонов
-   как шаблоны влияют на мониторинг хоста

5.  Изучите раздел **Latest data**.

-   найдите метрики одного из хостов
-   посмотрите последние значения метрик
-   определите период обновления данных

6.  Изучите работу с проблемами.

В разделе **Problems**:

-   найдите текущие проблемы
-   изучите уровень критичности (Severity)
-   посмотрите детали проблемы
-   выполните Acknowledgement проблемы (если возможно)

7.  Изучите фильтрацию и поиск.

Выполните поиск:

-   по имени хоста
-   по тегам
-   по группе хостов

------------------------------------------------------------------------

## 📊 В отчёте необходимо отразить

1.  Краткое описание структуры интерфейса Zabbix.

2.  Скриншоты:

-   списка хостов
-   параметров выбранного хоста
-   применённых шаблонов
-   раздела Latest data
-   раздела Problems

3.  Краткое описание следующих объектов:

-   Hosts
-   Templates
-   Items
-   Triggers
-   Tags
-   Macros

4.  Выводы:

-   как устроена структура мониторинга Zabbix
-   как связаны hosts, templates, items и triggers
-   какие инструменты интерфейса помогают анализировать проблемы

## 🛡 Требования к оформлению (анти‑плагиат)

1.  Каждый скриншот должен содержать ФИО студента, группу и номер
    работы:
    echo "ФИО, гр.XXX, ЛР7.6"

2.  На одном из скриншотов должны быть видны команды:
    whoami
    hostname

3.  За использование чужих скриншотов работа может быть оценена в 0
    баллов.
